## 2. 의제 2 — RULE_7 양방향 실패 대응

### 잠정안: 옵션 D (모니터링 임계값)

- RULE_7 자체는 유지
- 운영 단계에서 발동률 모니터링
- 임계 초과 시 알람
- 결정 자체는 데이터 누적 후로 미룸

### 운영 모니터링 3지표 (구현 필수)

1. **RULE_7 발동률** (rule_hits에 RULE_7 포함 비율)
2. **T1 trigger value** (T1 트리거 발동 후 majority 결과가 1회 결과와 다른 비율)
3. **T1 → needs_human 분기 비율**

### 거절한 옵션

- **A. 제거**: 의제 1 v0.95 폐기 트리거. 무작위 발동 11%만으로 카테고리 폐기는 과잉 대응

- **C. 분할 (7a / 7b)**: 표본 부족. 분할 정당화 데이터 0건

## 8. 본 세션 학습 메모 (2건)

### 학습 1: 분석 도중 발견되는 결함은 분석 자체의 가치 신호

의제 1 분석 중 어제 v0.9의 트리거 3 작동 불가 결함 발견. 이건 어제 작업의 실패가 아니라 분석이 제대로 작동했다는 증거. 24시간 휴식 후 재검토의 가치를 입증한 사례.

→ **의제별 분석 시 "어제 결정 그대로 유지"가 기본값이면 분석할 필요 없음**. 재검토에서 caveat 또는 결함 1건 이상 발견해야 분석 효용 있음. 만약 분석 결과가 "어제와 동일"로 끝나면 그건 분석 부족 신호.

### 학습 2: 표본 부족 상황의 정직한 잠정안 = "보류"

의제 2에서 옵션 D(모니터링 + 결정 보류) 잠정 권고는 분석 결과의 "회피"가 아니라 데이터 부족을 인정한 정직한 결정. 27회 표본으로 카테고리 폐기/재정의/분할 중 선택은 통계적으로 부당.

→ **향후 분석 시 표본 < 30~50회면 "결정 보류 + 모니터링 지표 정의"를 첫 번째 잠정안으로 검토**. 무리한 결정보다 결정 회피 자체가 권고가 되는 케이스 인지.

→ CLAUDE_MASTER.md v0.2 갱신 시 "데이터 기반 의사결정 표본 임계값" 항목 등재 검토.
